STL

stl - стандартная шаблонная библиотека, 
представляет собой контейнер итераторов, 
алгоритмов и прочих сущностей для работы с данными.

Ключевым понятием Stl является Container.

Container - это объект, который организует хранение других объектов в памяти.
Каждый контейнер для хранения объектов использует ту или иную структуру данных.(Массив,Список,Бин.Дерево)
Некоторые контейнеры комбинируют в себе разные структуры данных.
У каждого контейнера есть свой набор итераторов.

Каждый контейнер объявлен в своём отдельном заголовочном файле,
следовательно чтобы воспользоваться соответствующим контейнером, 
нужно подключить соответствующий заголовок.

Абсолютно все сущности в STL являются шаблонными.

Контейнеры бывают: Линейные и Ассоциативные.

Линейный.
Sequence containers: 
								array, 
								vector, 
								deque, 
								list, 
								forward_list;

Ассоциативный.
Associative containers: 
								map,
								set, 
								multimap,
								multiset;

Также существует контейнер Адаптер.
Adaptor containers: 
								stack, 
								queue, 
								priority_que;

Unordered associative containers: 
								unordered_map, 
								unordered_set, 
								unordered_multimap, 
								unordered_multiset;





Обработка исключений.

Исключительная операция - это ошибка на этапе выполнения,
и большинство этих ошибок можно не допустить на этапе написания кода,
но далеко не все, если исключения невозможно предотвратить на этапе написания кода,
то его обязательно нужно обработать, а для того чтобы обработать исключение, 
код который потенциально может его сгенерировать помещают в блок try, 
у которого есть как минимум один обработчик блок catch.

Блок catch  это самая обычная функция с одним параметром.
У блока try может быть сколько угодно обработчиков, 
но не может быть двух обработчиков с параметром одного типа.

Каждый обработчик обрабатывает только один тип исключений.

Для того чтобы бросить исключение используется ключевое слово throw.
